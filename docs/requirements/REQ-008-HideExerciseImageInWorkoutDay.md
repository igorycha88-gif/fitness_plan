# REQ-008: Скрыть изображение упражнения в дне тренировки

**Status**: New  
**Priority**: Medium  
**Complexity**: Simple  
**Created**: 2026-02-17  
**Updated**: 2026-02-17  

## Description
Убрать отображение изображения упражнения в карточке дня тренировки на экране плана. Изображение должно отображаться только при переходе на экран детализации упражнения.

## User Story
Как пользователь,  
Я хочу видеть компактный список упражнений в дне тренировки без картинок,  
Чтобы быстрее просматривать план тренировки и не тратить время на прокрутку длинного списка.

## Acceptance Criteria
- [ ] Изображения упражнений не отображаются в компоненте ExerciseRow
- [ ] Изображения упражнений не отображаются в AutoPlanSection
- [ ] Изображения упражнений не отображаются в UserPlanSection
- [ ] Изображения корректно отображаются на экране ExerciseDetailScreen при переходе
- [ ] Изображения корректно отображаются на экране ExerciseGuideScreen
- [ ] Изображения корректно отображаются на экране ExerciseLibraryDetailScreen
- [ ] Компактность списка упражнений визуально улучшена

## Implementation Notes

### Recommended approach
Удалить блок кода отображения изображения в функции `ExerciseRow` (HomeScreen.kt:871-883):

```kotlin
// УДАЛИТЬ ЭТОТ БЛОК:
exercise.imageRes?.let { resName ->
    val resId = context.resources.getIdentifier(resName, "drawable", context.packageName)
    if (resId != 0) {
        Image(
            painter = painterResource(id = resId),
            contentDescription = exercise.name,
            modifier = Modifier
                .fillMaxWidth()
                .height(150.dp)
                .padding(top = 8.dp)
        )
    }
}
```

### Affected locations
1. **HomeScreen.kt:871-883** - функция `ExerciseRow` - удалить блок отображения изображения
2. **ExerciseDetailScreen.kt:251** - оставить без изменений (изображение здесь нужно)
3. **ExerciseGuideScreen.kt:75-77** - оставить без изменений
4. **ExerciseLibraryDetailScreen.kt:74-76** - оставить без изменений

### Potential risks
- Пользователи могут не сразу узнавать упражнения без визуальной подсказки
- Низкий риск: изображения доступны на экране детализации

### Performance considerations
- Положительное влияние: уменьшение использования памяти при прокрутке списка
- Улучшение производительности при отображении длинных списков упражнений

### Security considerations
- Нет влияния на безопасность

## Validation Plan

### Unit tests
- Существующие UI тесты HomeScreen должны проходить
- Проверить что ExerciseRow отображает корректно без изображения

### Manual testing
1. Открыть приложение
2. Перейти на экран "План"
3. Развернуть день тренировки
4. Проверить что упражнения отображаются без изображений
5. Нажать на упражнение
6. Проверить что на экране детализации изображение отображается
7. Проверить для обоих типов планов: Автоматический и Мой план

### Edge cases
- Упражнение без изображения (imageRes = null)
- Упражнение с некорректным именем ресурса
- План с большим количеством упражнений (проверить производительность прокрутки)

### Error scenarios
- Нет применимо (только удаление кода)

## Metrics
- Высота списка упражнений уменьшена на ~150dp на каждое упражнение
